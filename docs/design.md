# NoiseGuard — 楼上噪音智能监测系统 设计文档

> 最后更新: 2026-03-29

---

## 一、项目目标

利用 ERAZER USB 全向麦克风，实时监听房间声音，通过**频率分析**自动区分：

- 🔴 **楼上冲击噪音**：脚步、拖家具、摔东西、跳跃（特征：低频为主 20-200Hz，突发短促）
- 🟢 **自家正常声音**：说话、音乐、做饭、开关门（特征：中高频为主 300Hz+，持续时间较长）
- ⚪ **安静**：背景噪声水平以下

检测到楼上噪音后自动记录 + 报警，生成统计数据作为证据。

## 三、检测原理

### 3.1 为什么频率分析能区分楼上噪音？

楼上的撞击通过楼板传到你房间时，高频成分被混凝土大量衰减，只有低频（<200Hz）能穿透，
表现为"咚咚咚"的闷响。而你自己在房间里说话、放音乐，高频成分完整保留。

频谱特征对比：

```
            楼上脚步           自家说话           自家音乐
20-200Hz    ████████████       ██                 ████
200-500Hz   ██                 ████████           ██████
500-2000Hz  ▏                  ██████████████     ████████████
2000Hz+     ▏                  ████████           ██████████████
```

### 3.2 检测算法

```
音频输入 (16kHz, mono)
    │
    ├── 1. 能量门限检测 ──→ 低于阈值 → 判定"安静"
    │
    ├── 2. FFT 频谱分析 (2048-point)
    │     ├── 低频能量 E_low  = sum(20-200Hz)
    │     ├── 中频能量 E_mid  = sum(200-2000Hz)
    │     └── 高频能量 E_high = sum(2000-8000Hz)
    │
    ├── 3. 频带比值判定
    │     ├── R = E_low / (E_mid + E_high + ε)
    │     ├── R > 阈值(默认2.0) → 低频主导 → 疑似楼上噪音
    │     └── R ≤ 阈值 → 自家声音
    │
    ├── 4. 冲击特征检测（辅助）
    │     ├── 起攻时间 < 50ms → 突发冲击（脚步特征）
    │     └── 连续低频事件间隔 0.3-1.5s → 脚步节奏
    │
    └── 5. 输出判定结果 + 置信度
```

### 3.3 自适应校准

首次启动时，在安静环境录制 5 秒建立**背景噪声基线**（底噪水平）。
之后的检测都基于"超出底噪多少 dB"来判断，适应不同环境。

## 四、功能清单

### 核心功能

| 功能           | 说明                                          |
| -------------- | --------------------------------------------- |
| 实时频谱监控   | 波形图 + 频谱图，低频高亮显示                 |
| 噪音事件检测   | 自动识别楼上噪音事件，标注开始/结束时间       |
| 事件录音       | 检测到噪音自动保存录音片段（前后各缓冲 3 秒） |
| 报警提示       | 弹窗 + 声音提示（可选）                       |
| "我在活动"按钮 | 手动标记自己正在制造噪音，避免误报            |

### 统计功能

| 功能      | 说明                                               |
| --------- | -------------------------------------------------- |
| 事件日志  | 时间、持续时长、最大强度、类型                     |
| 日/周统计 | 按时段统计噪音分布（如"22:00-01:00 共发生 47 次"） |
| 导出报告  | CSV/PDF 导出，可作为投诉证据                       |

### 设置

| 功能       | 说明                           |
| ---------- | ------------------------------ |
| 设备选择   | 从可用设备中选择 ERAZER 麦克风 |
| 灵敏度调节 | 调整检测门限                   |
| 频率范围   | 自定义低频/中频分界点          |
| 录音管理   | 自动清理旧录音（保留天数设置） |
| 校准       | 重新采集背景噪声基线           |

## 五、技术方案

### 5.1 技术栈

| 组件     | 技术          | 说明                  |
| -------- | ------------- | --------------------- |
| 语言     | Python 3.10+  | 生态最佳              |
| 音频采集 | sounddevice   | 跨平台，低延迟        |
| 信号处理 | numpy + scipy | FFT、滤波、频谱分析   |
| GUI      | PyQt5         | 成熟桌面框架          |

### 5.3 核心数据流

```
┌──────────────┐
│ sounddevice  │  16kHz mono, float32
│ InputStream  │  每帧 1024 samples (64ms)
└──────┬───────┘
       │ audio_frame signal
       ▼
┌──────────────┐
│   Analyzer   │  FFT → 频带能量 → 分类
│              │  输出: {type, low_energy, mid_energy, high_energy,
│              │         ratio, is_impact, confidence}
└──────┬───────┘
       │ analysis_result signal
       ▼
┌──────────────┐
│   Detector   │  状态机：管理事件生命周期
│              │  安静 → 触发 → 确认中 → 事件进行中 → 冷却 → 安静
│              │  输出: event_start / event_end / event_update
└──────┬───────┘
       │
       ├──→ Recorder (保存录音)
       ├──→ AlertManager (报警)
       ├──→ Database (记录)
       └──→ UI (更新显示)
```

### 5.4 检测器状态机

```
         音量 < 门限
    ┌────────────────────┐
    │                    │
    ▼                    │
 [安静] ──低频超标──→ [触发]
                         │
                    连续 N 帧确认
                         │
                         ▼
                    [事件进行中] ──低频恢复正常──→ [冷却]
                         │                           │
                    持续更新事件信息              等待 2 秒
                                                     │
                                              再次触发? ──是──→ [事件进行中]
                                                     │
                                                     否
                                                     │
                                                     ▼
                                                 [事件结束]
                                                  → 保存録音
                                                  → 写入数据库
                                                  → 更新统计
```

### 6.2 频谱图说明

频谱图是核心展示元素：

- X轴: 频率 (20Hz - 8000Hz，对数刻度)
- Y轴: 能量 (dB)
- **20-200Hz 区域**用红色背景高亮标注为"楼上噪音区域"
- **200Hz 以上**用蓝色标注为"自家声音区域"
- 实时频谱曲线叠显示在上面
- 当低频能量占比高时，红色区域变亮/闪烁
